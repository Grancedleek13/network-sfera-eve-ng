# План адресации

Внутренняя адресация базируется на сети `192.168.60.0/24`, которая делится на несколько подсетей с запасом по количеству хостов.

## VLAN10 – Маркетинг

- Подсеть: `192.168.60.0/26`
- Маска: `255.255.255.192`
- Диапазон хостов: `192.168.60.1–192.168.60.62`
- Шлюз: `192.168.60.1` (Router e0/0.10)

Пример хоста:
- PC1: `192.168.60.10/26`, GW `192.168.60.1`

## VLAN20 – HR

- Подсеть: `192.168.60.64/26`
- Маска: `255.255.255.192`
- Диапазон хостов: `192.168.60.65–192.168.60.126`
- Шлюз: `192.168.60.65` (Router e0/0.20)

Пример хоста:
- PC2: `192.168.60.70/26`, GW `192.168.60.65`

## VLAN30 – Серверная

- Подсеть: `192.168.60.128/27`
- Маска: `255.255.255.224`
- Диапазон хостов: `192.168.60.129–192.168.60.158`
- Шлюз: `192.168.60.129` (Router e0/0.30)

Пример хоста:
- WEB-SRV: `192.168.60.130/27`, GW `192.168.60.129`

## Внешняя сеть

- Линк Router–ISP: `203.0.113.4/30` (пример)
  - Router: `203.0.113.5`
  - ISP: `203.0.113.6`
- Публичный адрес для NAT: `203.0.113.5`.
