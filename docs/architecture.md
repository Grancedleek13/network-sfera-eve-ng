# Архитектура сети «Сфера»

Сеть моделирует офис небольшой компании с тремя логическими сегментами:

- отдел маркетинга (VLAN10);
- отдел HR (VLAN20);
- серверная (VLAN30).

Все рабочие станции и серверы подключены к L2-коммутатору **SW1**.  
Между VLAN маршрутизирует **Router** по схеме *router-on-a-stick* через dot1q‑транк.

Через узел **ISP** эмулируется подключение к Интернету и внешняя сторона для проверки NAT/DNAT.

Ключевые моменты архитектуры:

- каждый отдел сидит в своём VLAN → широковещательные домены разделены;
- между отделами настроена L3‑изоляция (ACL), но доступ к серверной VLAN разрешён;
- NAT позволяет всем хостам выходить в Интернет с одного публичного адреса;
- веб‑сервер в VLAN30 опубликован наружу через DNAT.
