# Результаты тестирования

## 1. Проверка связности внутри VLAN

С хостов:
- PC1 пингует свой шлюз `192.168.60.1`;
- PC2 пингует свой шлюз `192.168.60.65`;
- WEB-SRV пингует свой шлюз `192.168.60.129`.

## 2. Проверка ACL (изоляция отделов)

- С PC1 (VLAN10) нет ответа на ping `192.168.60.70` (PC2, VLAN20).
- С PC2 (VLAN20) нет ответа на ping `192.168.60.10` (PC1, VLAN10).

При этом:
- PC1 пингует WEB-SRV `192.168.60.130`.
- PC2 пингует WEB-SRV `192.168.60.130`.
- WEB-SRV пингует PC1 и PC2.

## 3. Проверка NAT/Интернета

- PC1/PC2/WEB-SRV пингуют адрес провайдера (например, `203.0.113.6`).
- На Router видны NAT‑трансляции (`show ip nat translations`), где внутренние адреса 192.168.60.x отображаются на 203.0.113.5.

## 4. Проверка публикации веб‑сервера

- Со стороны "Интернета" (через ISP) доступен `203.0.113.5:80`.
- На Router настроен статический NAT для проброса порта 80 на `192.168.60.130`.

Эти проверки демонстрируют, что:
- сегментация и изоляция VLAN работают;
- маршрутизация и NAT настроены корректно;
- серверная VLAN доступна из отделов и опубликована наружу.
